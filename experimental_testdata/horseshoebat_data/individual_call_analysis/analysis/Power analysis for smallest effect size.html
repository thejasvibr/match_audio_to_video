What this notebook will do¶

This notebook will try to estimate the power our kernel-density based overlap method has for a variety of effect sizes. This is not a post-hoc power analysis because I am not using the effect-size estimated from the data. There are good reasons not to do post-hoc power-analyses mainly because they do not really reveal the actual power of a test [1,2]. What I will do instead is to run a power analysis to see if the method could pick up the smallest reasonable effects for each parameter of interest.

Luckily for us, we also have a previously published study (Fawcett et al. 2015)[3] on another rhinolophid's echolocation behaviour as single individuals and pairs. [3] will be used to set the minimum effect sizes, and calculate the power for these minimum effect sizes.

Calculating 'power' and making the simulated data¶

The power is the rate at which the statistical test detects true positives. Here 'true positives' are those cases where the comparison of two groups is different by the minimum detectable effect (MDE). I will also implement simulated data where the two groups are from the same distribution to assess the rate of false positives.

For our particular case, the observed percentile of the two distributions is the measure of interest. When the distributions (single and multi bat call parameters) are similar, the overlap percentile will be relatively high (because the observed overlaps are close to the shuffled data overlaps). When the distributions are different, the observed percentiles will be low in comparison to the shuffld data overlaps. Thus the test 'detects' a difference when the observed overlap is low.

How much power is 'enough'?¶

Since this power analysis is being done after the experiments, and not before, the question is how will the resulting power calculations affect our interpretation of the data. We clearly cannot reject any tests which are lower than the typical 0.8 power threshold.

If we do see an effect in our final results, we need to weigh it with the observed power for those parameters. If we do not see differences between our single and multi bat groups, we need to understand how likely it is that the results are a true negative, or a false negative.

How much power is enough really depends on the consequences of the interpretation [4].In our case, it is better to have Type II (false negatives) results rather than Type 1 (false positives), and so even if our power is low - we could still gain a lot by reporting the results as they are. The fact that bats do not adjust their call parameters may actually be a point of discussion.

Setting the minimum expected effect sizes:¶

Luckily for us we have a previous publication with rhinolophid bats [3] which can inform us about the minimum expected effect sizes. Of course, Fawcett et al. 2015 [3] studied the echolocation changes in a different species, (Rhinolphus capensis) , and the actual effect sizes may actually be lower in our species. To avoid wild speculation, I'm just going to estimate effect sizes from [3], and then proceed to see what the power of our tests are.

  • ### No difference in (CF) peak frequency
  • ### 1.25-1.9 X increase in tFM duration in pairs
  • ### 0.74 X times shorter calls in pairs
  • ### lower minFM in FM sweeps. Drop in minimum frequency (-10dB) between 3-5 kHz
  • ### A drop in source level in pairs, but this could also be attributed to changes in beam shape for instance.

def include_measurement_accuracy(X, measurement_accuracy):
    '''
    '''
    return np.array([return_rounded_output(each, measurement_accuracy) for each in X])
        
def return_rounded_output(each, measurement_accuracy):
    remainder = np.remainder(each, measurement_accuracy)
    if remainder/measurement_accuracy < 0.5:
        return measurement_accuracy*int(each/measurement_accuracy)
    elif remainder/measurement_accuracy >=0.5:
        return measurement_accuracy*(int(each/measurement_accuracy)+1)

        # need to parametrise this stuff to the experimental data

                                                                                      ,num_shuffles=N_shuffles)
        observed_percentile = shuffle_overlap.percentile_score_of_value(data_overlap, shuffled_overlaps)

        result_significant = observed_percentile < alpha
        results_significant.append(result_significant)

fm_duration_accuracy = 0.1*10**-3 # assumed duration measurement accuracy 

fm_change = [1, 0.75, 1.25]

all_obs_pctile = []
for change in fm_change:
    for i in tqdm(range(num_trials)):
        # need to parametrise this stuff to the experimental data
        solitary_bat_pf = np.random.normal(mean_tfm_durn, sd_tfm_durn, single_sample_size) # in ms
        solitary_bat_pf = include_measurement_accuracy(solitary_bat_pf, fm_duration_accuracy)

        group_bat_pf = np.random.normal(mean_tfm_durn*change,sd_tfm_durn, group_sample_size) # in ms
        group_bat_pf = include_measurement_accuracy(group_bat_pf, fm_duration_accuracy)

        data_overlap, outputs, shuffled_overlaps = shuffle_overlap.overlap_w_shuffling(solitary_bat_pf,
                                                                                       group_bat_pf
                                                                                      ,num_shuffles=N_shuffles)
        observed_percentile = shuffle_overlap.percentile_score_of_value(data_overlap, shuffled_overlaps)

        # The overlap is always expected to be low if the distributions are different - and so check for 
        # this.
        result_significant = observed_percentile < alpha
        results_significant.append(result_significant)
        all_obs_pctile.append(observed_percentile)
    power = sum(results_significant)/num_trials
    print(power)

overall_power = sum(results_significant)/len(fm_change)*num_trials
